vortex_backtest 指南

📊 看板 · 🔌 Swagger（交互式 API） · ReDoc

它是什么

独立 HTTP 会话式回测/账户回放服务（A 股分钟级）。交互模型： 建会话 → 按模拟时钟逐步 advance（提交委托+推进）→ close 出报告。 服务端控 sim_time、按 as_of 强制 point-in-time 取数（防未来函数）； T+1 / 涨跌停 / 分板手数 / 费用由规则内核强制。

七步开闭环

① POST /accounts                建账户 {account_id, initial_cash}
② POST /sessions                开会话 {account_id, level, start_date, end_date, universe}
③ POST /sessions/{id}/advance   买入+推进 {request_id, to, orders:[{symbol, side:1, quantity, …}]}
④ POST /sessions/{id}/advance   T+1 后卖出
⑤ POST /sessions/{id}/advance   {to:"end"} 推进到期末
⑥ POST /sessions/{id}/close     关闭出报告
⑦ GET  /sessions/{id}/summary   汇总（或一条命令: scripts/backtest_roundtrip.sh）

两条取数路（口径不同）

路触发口径分红
data 网关（推荐）配 VORTEX_DATA_URL+tokenRAW 不复权真实价除权日显式入账（真实账户）
本地直读（回退）配 VORTEX_WORKSPACEqfq 前复权不入账（已吸进价）

两路总收益近似一致，现金流/估值数值不同，不混用对账。

鉴权

写接口 fail-closed：配了 VORTEX_BACKTEST_TOKEN 须带 Authorization: Bearer 或 X-Auth-Token；未配时仅本机回环放行。 容器内监听 0.0.0.0 属非回环——容器部署必须配 token（本仓 .env）。
